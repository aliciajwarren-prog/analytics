LibLynx Design System

Input

Input components allow users to enter and edit text. They support various states, styles, and can include additional elements like tags or icons.

Input States

Focus

Usage Guidelines

When to Use

  • When users need to enter free-form text
  • For search queries, form fields, and data entry
  • When the input format is flexible or unknown
  • For single-line text input

When Not to Use

  • When there are predefined options (use dropdown instead)
  • For multi-line text (use textarea instead)
  • When input format is highly constrained (use specialized inputs)
  • For binary choices (use radio buttons or checkboxes)

Accessibility

  • Provide clear labels for all inputs
  • Use placeholder text sparingly and as hints only
  • Ensure sufficient color contrast for all states
  • Provide clear error messages and validation feedback
  • Support keyboard navigation and screen readers
